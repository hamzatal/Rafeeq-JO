# 04 — كتالوج المزايا (تطوير / إضافة / تأجيل)

> قرارات المزايا بناءً على دراسة السوق (50+ تطبيق: Uber, Bolt, inDrive, Careem, Grab, DiDi...) وتوجيهاتك.
> المبدأ الاستراتيجي: **تميّز في النواة، لا تنافس على العدد.** رفيق كان يعاني «تضخّم مزايا» (32 موديول).

---

## 1) 🟢 النواة (نتقنها أولاً — أولوية قصوى)

| الميزة | الحالة | العمل |
|--------|--------|-------|
| رحلة الطلب → التتبّع → الدفع | موجودة | تصميم Stitch «خريطة-أولاً» + خريطة تفاعلية |
| التسعير بالمسافة (GPS) | **ناقصة (سعر ثابت)** | محرّك تسعير جديد — [02-PRICING-ZONES](./02-PRICING-ZONES.md) |
| عمولة المنصة + أرباح الكابتن | موجودة (تُكمَّل بشاشة/معاينة) | ملخّص أرباح + معاينة قبل القبول |
| الاشتراك الجامعي (سعر موحّد) | موجودة (تُربط بالمسافة) | سعر موحّد جامعة↔بيت |
| المحفظة + CliQ + GPT Vision | موجودة | ربط مفاتيح حقيقية |

## 2) 🟢 مزايا نطوّرها (موجودة لكن ناقصة)

| الميزة | التطوير |
|--------|---------|
| تتبّع الرحلة | **«شارك رحلتي»** برابط حيّ للعائلة + PIN تحقّق عند الركوب |
| SOS | مشاركة موقع تلقائية + **كشف توقّف/انحراف مسار (RideCheck)** |
| الملف الشخصي | شاشة تعديل ملف كاملة (ناقصة حالياً) |
| الفواتير | إيصال/فاتورة PDF موحّدة للطرفين |

---

## 3) ➕ مزايا نضيفها (تمييز تنافسي)

1. **وضع أمان الطالبات / خيار كابتن-أنثى** — تمييز ضخم في سوق الجامعات الأردني (ثقة الأهل).
2. **الكاربول (طالبات بسياراتهن):** الطالبة تسجّل سيارتها وتقلّ طالبات في مسارها للجامعة وتربح — يوسّع الأسطول بتكلفة صفر ويعزّز أمان الطالبات. راجع [00-VISION §9](./00-VISION.md).
3. **الرحلات المجدولة المتكرّرة** (اشتراك صباح الجامعة) بواجهة واضحة — نمط الاستخدام اليومي.
4. **مشاركة الأجرة (Split fare)** بين طلاب نفس الوجهة.
5. **ETA واعٍ بالسياق المحلي** (أوقات المحاضرات/الصلاة) — لمسة إقليمية ذكية.

---

## 4) 📢 المساحات الإعلانية (جديد — مطلوب)

تجهيز البنية للإعلانات من الآن (ربح مستقبلي)، بتصميم قويّ لا يزعج التجربة:

**أماكن مدروسة (Ad Slots):**
- **بانر الرئيسية** (تحت الشيت السفلي / بطاقة قابلة للتمرير) — الأبرز.
- **شاشة انتظار المطابقة** (أثناء البحث عن كابتن) — وقت انتباه عالٍ.
- **ملخّص نهاية الرحلة / الإيصال** — تفاعل مضمون.
- **سجل الرحلات (بطاقة كل عدة عناصر)** — أصلية غير مزعجة.

**التصميم:** بطاقة إعلان بنفس هوية Stitch (حواف 16px، ظل ناعم)، وسم «إعلان» واضح، أبعاد ثابتة تمنع القفز، دعم صورة + عنوان + CTA + رابط.

**لوحة التحكم (إدارة كاملة):** قسم «الإعلانات» يتيح: إنشاء/تعديل حملة، اختيار المكان (slot)، الجدولة (بداية/نهاية)، الاستهداف (مدينة/جامعة)، الأولوية، وإحصاءات (ظهور/نقر). البنية مستوحاة من نمط **Server-Driven UI** (بطاقات يرسلها الخادم) لتغيير الإعلانات بلا تحديث التطبيق.

## 5) 🤖 مزايا الذكاء الاصطناعي (GPT API — اعتماد كامل)

| الميزة | الوصف | الحالة |
|--------|-------|--------|
| مساعد رفيق | مساعد داخل التطبيق (طلب/اشتراك/توجيه) | بنية موجودة + fallback |
| تحقّق الدفع (Vision) | قراءة صورة تحويل CliQ | موجودة |
| ذكاء المسارات | أفضل تجمّع/مسار/وقت | موجودة (تُفعّل) |
| مراقبة الاحتيال/الأمان | كشف الأنماط الخطرة | موجودة |
| تحليل الشكاوى/الدعم | تصنيف + خطورة + حلول | موجودة |
| مطابقة المفقودات | ربط مفقود/موجود | موجودة |
| تحليلات الإدارة | رؤى ذكية | موجودة |

> كلها عبر `GptClient` مع `NullGptClient` كـ fallback آمن. المرحلة 6: ربط مفتاح GPT حقيقي + ضبط الميزانية (AiBudget موجود).

## 6) 🟡 مزايا نؤجّلها/نبسّطها (تخفيف التضخّم — بلا حذف كود)

| الميزة | القرار |
|--------|--------|
| التبادل (Exchange) | إخفاء خلف feature flag (قيمته غير واضحة لتطبيق نقل) |
| المفقودات (LostFound) | دمجها ضمن «الدعم» بالواجهة بدل موديول بارز |
| المكافآت | الإطلاق بـ «دعوة صديق» فقط أولاً |
| **«اقترح سعرك» (تفاوضي)** | **مؤجّلة بطلبك — غير مهمّة حالياً** |

> التأجيل = إخفاء بالواجهة عبر flag، **لا حذف للسكيما/الكود** (احتراماً لمبدأ «بلا كسر»). عند الرغبة نُعيد التفعيل بسهولة.
