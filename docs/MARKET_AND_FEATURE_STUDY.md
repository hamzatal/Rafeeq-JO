# دراسة السوق والمزايا والتصميم لمنصّة رفيق (RFQ-291)

> تقرير بحثي تفصيلي: دراسة 50+ تطبيق نقل عالمي ومحلي، مراجعة مزايا رفيق (تطوير/إضافة/حذف)، وتشخيص مشاكل التصميم/اللاي-آوت/الأيقونات مع حلول مستوحاة من أفضل الممارسات العالمية.
> *ملاحظة توافقية: كل المعلومات الخارجية أُعيدت صياغتها من مصادرها العامة مع روابط الإسناد.*

---

## أولاً: رأيي الصريح بالمشروع (كمهندس درس كل سطر)

رفيق **مشروع قوي جداً وليس مبتدئاً**: معماريته نظيفة (Modular Monolith · 32 موديول · 65 جدول · 166 اختبار)، ومنطق الأعمال حقيقي ومختبَر (محفظة بأقفال، دفع CliQ بذكاء اصطناعي، مطابقة، أمان). المشكلتان اللتان أعاقتاه: **فوضى الهوية البصرية** (حُلّت بتوحيد Onyx) و**ثغرات أمان** (عولجت). 

**لكن** — بصراحة — المشروع يعاني من **«تضخّم المزايا» (Feature Bloat):** 32 موديول تحاول تقليد «سوبر-آب» (نقل + طرود + تبادل + مفقودات + مكافآت + مساعد) قبل إتقان النواة. المنافس الأنجح محلياً (inDrive) كسب السوق بـ **ميزة واحدة مميّزة** لا بـ 30 ميزة متوسطة. **توصيتي الاستراتيجية: تميّز في النواة، لا تنافس على العدد.**

## ثانياً: دراسة السوق — ماذا تفعل أفضل 50+ تطبيق؟

### أ) اللاعبون المدروسون (عالمي + إقليمي + محلي)
Uber · Lyft · Bolt · inDrive · DiDi · Grab · Ola · Cabify · Gojek · Yango · Careem · Jeeny · Gett · FreeNow · BlaBlaCar · Blacklane · Alto · Curb · Wingz · Co-op Ride — إضافةً إلى عشرات القوالب على Dribbble ومستودعات مفتوحة المصدر. ([قائمة بدائل Uber](https://www.apurple.co/uber-alternatives/) · [أفضل تطبيقات النقل 2026](https://mobisoftinfotech.com/resources/blog/list-of-popular-rideshare-apps-globally))

### ب) نماذج التسعير — الدرس الأهم
- **inDrive (تسعير تفاوضي):** الراكب يقترح السعر والكابتن يقبل/يعرض مقابلاً — إحساس «عدل وإنساني». عمولته ~10% مقابل ~42% لأوبر، وهذا سرّ جذبه للكباتن. النقطة الذكية: **رقم مقترَح واحد (anchor)** يمنع «شلل الحقل الفارغ» ويجعل الحرية قابلة للاستخدام. ([التحليل](https://www.ainvest.com/de/news/indrive-bidding-model-alpha-leak-ride-hailing-growth-engine-2603/) · [تفكيك واجهة inDrive](https://uxsnaps.beehiiv.com/p/ui-breakdown-48-indrive-lets-you-set-your-own-fare-one-recommended-number-makes-that-actually-work)) — *أُعيدت الصياغة.*
- **Uber (خوارزمي):** السعر تحسبه الخوارزمية، توفّر عالٍ وتجربة سلسة. ([مقارنة](https://fangwallet.com/2025/09/20/uber-vs-indrive-rideshare-app-comparison/))
- **الخلاصة لرفيق:** رفيق يستخدم تسعيراً ثابتاً + عمولة. **مقترح تمييزي:** أضِف **«اقترح سعرك»** كخيار للرحلات الطويلة/خارج الذروة — يناسب حسّاسية الطلاب للسعر ويميّزنا عن كريم.

### ج) أنماط التصميم السائدة (التي يجب أن نتبنّاها)
1. **خريطة كاملة + شيت سفلي (Bottom Sheet):** الشاشة الرئيسية = خريطة، والإجراءات في شيت منزلق — معيار عالمي (أوبر/بولت). ([أنماط Dribbble](https://dribbble.com/tags/ride-app))
2. **زر إجراء واحد مهيمن:** «إلى وين؟» أو «اطلب رحلة» يهيمن بصرياً — لا تشتيت.
3. **واجهات مدفوعة بالخادم (Server-Driven UI):** أوبر تبني الشاشات من «بطاقات» (ActionCard) يرسلها الخادم — يتيح تغيير الواجهة بلا تحديث التطبيق. ([Uber ActionCard](https://www.uber.com/ca/en/blog/developing-the-actioncard-design-pattern/)) — *مفيد لرفيق مستقبلاً لعروض/إعلانات ديناميكية.*
4. **الوضع الداكن الحقيقي + نظام تصميم موحّد (Design Tokens):** أوبر لديها [Base design system](https://base.uber.com/). رفيق الآن على Onyx موحّد — خطوة صحيحة.

### د) المزايا التي تميّز الكبار (وما ينقص رفيق)
- **الأمان (أوبر معيار عالمي):** زر SOS يشارك الموقع الحيّ مع الشرطة، **«شارك رحلتي»** مع حتى 5 جهات موثوقة، **RideCheck** يكشف التوقّف المفاجئ/تغيّر المسار، تسجيل صوت، تحقّق PIN عند الركوب، وتخصيص ميزات أمان للنساء. ([التزام أوبر بالأمان](https://www.uber.com/us/en/safety/our-commitment/) · [أمان النساء](https://www.uber.com/ca/en/safety/womens-safety/)) — *أُعيدت الصياغة.*
- **كريم (سوبر-آب + AI):** تحقّق هوية الكابتن بالتعرّف على الوجه، و**ETA واعٍ بأوقات الصلاة والإفطار برمضان** (تميّز إقليمي ذكي). ([مقابلة كريم عن الـ AI](https://blog.careem.com/posts/ai-and-the-super-app-an-interview-with-careems-selim-turki)) — *أُعيدت الصياغة.*
- **Ola (تعدّد وسائط):** سيارات + تكاتك + دراجات + تأجير + رحلات بين المدن. ([المصدر](https://nextgrowthlabs.com/blog/best-ride-sharing-apps))

### هـ) مصادر مفتوحة للاستلهام التقني (GitHub)
- [`adrianhajdin/uber`](https://github.com/adrianhajdin/uber) — كلون أوبر Full-stack بـ **Expo + Postgres** (قريب جداً من ستاك رفيق — مرجع ممتاز للتدفّقات والخريطة).

## ثالثاً: مراجعة مزايا رفيق (تطوير / إضافة / حذف)

### 🟢 مزايا نطوّرها (موجودة لكن ناقصة)
| الميزة | التطوير المقترح |
|--------|------------------|
| تتبّع الرحلة | **«شارك رحلتي»** برابط حيّ للعائلة + PIN تحقّق عند الركوب (استلهام أوبر) — قيمة أمان ضخمة للطلاب. |
| SOS | ربط بالطوارئ + مشاركة موقع تلقائية + **كشف توقّف/انحراف مسار** (RideCheck) — الأساس موجود. |
| التسعير | إضافة **«اقترح سعرك»** للرحلات المرنة (استلهام inDrive) + تقدير أجرة واضح قبل التأكيد. |
| الملف الشخصي | **شاشة تعديل ملف** (ناقصة حالياً — الإعدادات = مظهر وروابط فقط). |
| الفواتير | إيصال/فاتورة PDF موحّدة داخل التطبيق للطرفين. |

### ➕ مزايا نضيفها (تمييز تنافسي)
1. **وضع أمان الطالبات / خيار كابتن-أنثى** — تمييز ضخم بسوق الجامعات الأردني (ثقة الأهل).
2. **الرحلات المجدولة المتكرّرة** (اشتراك صباح الجامعة) بواجهة واضحة — نمط الاستخدام اليومي الأساسي.
3. **«اقترح سعرك»** التفاوضي (اختياري) — يميّزنا عن كريم ويناسب ميزانية الطلاب.
4. **ETA واعٍ بالسياق المحلي** (أوقات المحاضرات/الصلاة) — لمسة ذكية إقليمية.
5. **مشاركة الرحلة مع الأصدقاء (Split fare)** بين الطلاب في نفس الوجهة.

### 🔴 مزايا نؤجّلها/نبسّطها (لتخفيف التضخّم — دون حذف كود)
- **تبادل (Exchange):** قيمته غير واضحة لتطبيق نقل → إخفاء خلف flag (مرشّح للحذف مستقبلاً).
- **مفقودات (LostFound):** دمجه ضمن «الدعم» بدل موديول مستقل بالواجهة.
- **المساعد الذكي الموجّه للمستخدم:** تأجيل (نُبقي فحص الاحتيال AI الخلفي) — كلفة تشغيل + مخاطر جودة.
- **المكافآت:** الإطلاق بـ «دعوة صديق» فقط.
> التأجيل = إخفاء بالواجهة عبر feature flag، **لا حذف للسكيما/الكود** (احتراماً لقرارك «بلا حذف مزايا»).

## رابعاً: تشخيص التصميم/اللاي-آوت/الأيقونات + الحلول

### المشاكل المكتشفة فعلياً في الكود
1. **أيقونات اتجاهية لا تنعكس في RTL (أُصلحت ✅ في RFQ-291):** `chevron-left`/`arrow-left`/`log-out`... كانت تشير للاتجاه الخاطئ بالعربي (زر رجوع يشير يسار). أضفت انعكاساً واعياً بالـ RTL في `Icon.tsx` للتطبيقين.
2. **حزم خطوط غير مستخدمة:** `@expo-google-fonts/tajawal` و`ibm-plex-sans-arabic` مثبّتتان بلا استخدام (النظام يستخدم Cairo فقط) → مرشّحة للإزالة (تقليل حجم).
3. **مكوّنات مكرّرة بين التطبيقين** (LiveMap 450 سطر، Feedback، Loader...) → توحيدها في `packages/shared` لاحقاً.
4. **اللاي-آوت الحالي جيد لكنه «قوائم» أكثر منه «خريطة-أولاً»** — المعيار العالمي هو الخريطة الكاملة + شيت.

### الحلول المستوحاة من البحث (خارطة تصميم للتنفيذ عند رفع التجميد)
- **الشاشة الرئيسية:** خريطة كاملة + شيت سفلي بـ «إلى وين؟» مهيمن + وجهات محفوظة (جامعتي/بيتي) — نمط أوبر/بولت.
- **الأيقونات:** الحفاظ على مجموعة واحدة متّسقة (Feather line-icons) بلا خلط مع إيموجي؛ حاويات أيقونات موحّدة الشكل.
- **الأزرار/المربّعات:** نظام حواف موحّد (من معرض الـ30 اخترنا لاحقاً) + زر أساسي واحد مهيمن لكل شاشة.
- **الخريطة:** ستايل خريطة داكن يطابق Onyx + خط مسار أزرق + دبابيس واضحة (انطلاق إنك/وجهة أزرق).
- **عملية الطلب/الإلغاء:** تدفّق من 3 خطوات واضحة (وجهة → تأكيد سعر → مطابقة/تتبّع) مع سياسة إلغاء ورسوم واضحة.

> **الخطوة العملية:** عند رفع تجميد التصميم، نطبّق اتجاهاً واحداً من [معرض الـ30 تصميم](./DESIGNS.md) على هذه الخارطة عبر التطبيقات الثلاث.

## خامساً: التوصية النهائية (أولويات التطوير)
1. **النواة أولاً:** أتقن رحلة الطلب→التتبّع→الدفع بتصميم خريطة-أولاً عالمي.
2. **الأمان كتمييز:** «شارك رحلتي» + وضع الطالبات + RideCheck — قيمة ثقة تكسب السوق الأردني.
3. **قصّ التضخّم:** أخفِ الثانوي خلف flags، وركّز الجودة على ما يُستخدم يومياً.
4. **لمسة تفاوض السعر:** اختياري، يميّزنا عن كريم بميزانية الطلاب.
5. **التصميم الموحّد:** اعتمد اتجاهاً واحداً من الـ30 وطبّقه بحرفية.
