# خطة العمل الشاملة — رفيق (Master Work Plan)

> تُحدَّث باستمرار. الحالة: التطوير جارٍ، الدفع المباشر على فرع `foundation/phase-0-1`.

## 0. أولاً: لماذا «كل خطوة يضرب معك شي»؟ (السبب الجذري + الحل)

معظم الأخطاء اللي واجهتها (`coupon_code`، `CHECK constraint: type`) **ليست في المنطق** — بل **قاعدة بياناتك المحلية تأخّرت عن مخطّط الكود**. لما نضيف عموداً أو قيمة enum جديدة، لازم تُطبّق على قاعدتك.

**الحل النهائي لهذا النوع كامل — نفّذه على جهازك:**
```bash
cd "…/Rafeeq-JO/backend"
git pull
php artisan migrate:fresh --seed     # يعيد بناء كل الجداول والقيود من المخطّط الحالي (يمسح بيانات التجربة)
# أو بدون مسح البيانات:  php artisan migrate
```
> **قاعدة ذهبية:** بعد كل `git pull` شغّل `php artisan migrate`. وعند أي خطأ «عمود/CHECK» شغّل `migrate:fresh --seed`.

**وقائياً من طرفي:** حوّلت عمود نوع معاملات المحفظة إلى string (يقبل أي نوع)، وأضفت اختبار HTTP يمرّ على تدفّق الرحلة كامل، وسأخصّص «جولة تثبيت» أجرّب فيها كل شاشة وزر (المرحلة E أدناه) قبل إكمال التصميم.

## 1. ما الذي أُنجز فعلاً (التصميم الجذري بدأ)
- **الهوية v6 (كحلي+ذهبي)** موحّدة عبر التطبيقات الثلاثة + نظام أزرار دلالي + نظام Toast/تأكيد + Loader متحرّك + skeletons.
- **أُعيد بناؤها بالكامل:** محفظة/شحن الطالب، رئيسية الطالب، تدفّق طلب/إلغاء الرحلة، الإشعارات، **معالج توثيق الكابتن**، أداء لوحة الإدارة (skeletons لكل الصفحات + إصلاح وميض الأيقونات + صورة الدخول).
- **الباك إند:** إصلاح انهيار طلب الرحلة، عكس الشحن الإداري، سباق قبول العرض، idempotency الدفع، إغلاق حالة الطلب، الذهاب‑إياب، طوابير للعمليات الثقيلة، واتساب OTP + احتياطي SMS، تقليم البيانات. **157 اختبار أخضر.**
- الخريطة: تعرض OSM المجانية تلقائياً لو فشل مفتاح Google (تقدر تعاين بلا مفتاح).

## 2. الخطة القادمة (بالتسلسل)

### المرحلة E — جولة تثبيت شاملة (الأولوية القصوى — تبدأ الآن)
**الهدف: صفر أخطاء تشغيل. أجرّب كل شاشة وكل زر وكل تدفّق بنفسي.**
- تشغيل الباك إند + إنشاء بيانات بذور، وتمرير آلي/يدوي على كل مسارات الـAPI (طالب/كابتن/أدمن) لاصطياد أي 500.
- فحص كل أعمدة enum/CHECK للتأكد من عدم وجود انحراف (تم فحص 55 موديل — 0 تعارض؛ سنُكمل التغطية).
- اختبارات HTTP للتدفّقات الحرجة (تمّ: تدفّق الرحلة؛ التالي: الاشتراك، الشحن، رفع الوثائق، الرحلة الكاملة كابتن↔طالب).
- **المخرج:** كل زر تضغطه يشتغل، ولا مفاجآت.

### المرحلة F — إكمال إعادة البناء الجذري شاشة‑بشاشة
الأساس والشاشات الأساسية جاهزة؛ يتبقّى تمشيط باقي الشاشات على نفس الهوية والجودة:
- **الطالب:** trips, checkout, subscriptions, payments, parcels, rewards, lost-found, exchange, support, chat, emergency, addresses, assistant, settings, onboarding, auth.
- **الكابتن:** offers, trip/[id], trips, earnings, invoices, withdraw, vehicle, chat, settings, onboarding, auth.
- **الأدمن:** تحديث كل صفحة على الكحلي+الذهبي (استبدال أصناف cyan→gold)، وتحديث التخطيط لكل الصفحات الـ26.
- تعميم الأنيميشن/التلميع البصري + إمكانية الوصول (التباين، أحجام اللمس).

### المرحلة G — التشغيل والإطلاق
- مفتاح Google Maps الحقيقي (تشغيلي) — أو ترقية لخريطة أصلية `react-native-maps`.
- بناء APK عبر EAS (الدليل: `docs/deployment/android-build.md`).
- تثبيت الاسم قانونياً (`docs/business/BRAND_NAMING.md`) → تحديث اللوجو/الحزم.
- تقوية أمان الإنتاج + مفاتيح CliQ/WhatsApp/FCM + تجريبي (Pilot) بجامعة واحدة.

## 3. التقدير الزمني (عمل فعلي)
- **المرحلة E (تثبيت):** ~2–3 جولات — تبدأ الآن.
- **المرحلة F (باقي الشاشات):** ~1–2 أسبوع (تمشيط منظّم، تطبيق تلو الآخر).
- **المرحلة G (تشغيل/إطلاق):** يعتمد على المفاتيح/الحسابات/التراخيص (يمشي بالتوازي).

## 4. المسؤوليات
- **أنا:** الكود + الاختبار + الدفع + التوثيق.
- **أنت:** `migrate` بعد كل pull، مفاتيح الخدمات، حسابات المتاجر، القرار القانوني للاسم.
