# 01 — خطة العمل الرئيسية والنهائية (Master Plan)

> **مصدر الحقيقة الوحيد للتخطيط.** كل الخطط/التقارير القديمة (PROGRESS, ROADMAP, REVAMP, REMEDIATION, AUDIT...) **حُذفت** واستُبدلت بهذا الملف.

## أ) القرارات الحاكمة (نهائية)

1. **التصميم:** اعتماد **Stitch** حصراً عبر التطبيقات الثلاثة، وحذف كل الهويات السابقة (7 محاولات). المرجع: [03-DESIGN-SYSTEM](./03-DESIGN-SYSTEM.md).
2. **الخط:** `IBM Plex Sans Arabic` (بديل Cairo نهائياً).
3. **الوضع واللغة:** يُدعم الوضعان (فاتح/داكن) واللغتان (عربي/إنجليزي)، والافتراضي **فاتح + عربي** لكل التطبيقات.
4. **الملاحة:** موبايل = Bottom Tabs (مع Rafeeq AI بالوسط) · إدارة = Sidebar يمين RTL.
5. **التسعير:** مبني على المسافة (GPS) + عمولة قابلة للضبط + سعر طالب موحّد. المرجع: [02-PRICING-ZONES](./02-PRICING-ZONES.md).
6. **الأمان:** إغلاق كل الثغرات والتحقّق الفعلي — بلا استثناء.
7. **AI:** اعتماد كامل على GPT API مع fallback آمن.
8. **الإطلاق:** إربد أولاً، والبنية جاهزة لتوسّع كل الأردن بكبسة زر.
9. **الإعلانات:** بنية + مساحات + إدارة من لوحة التحكم من الآن.
10. **«اقترح سعرك»:** مؤجّلة (غير مهمّة حالياً).

- [x] تصحيح الوثائق المتناقضة (Laravel 12/PHP 8.4، هوية Stitch، إلخ).

### المرحلة 1 — أساس التصميم (Design Foundation)
- اعتماد توكنز Stitch في `packages/shared` (ألوان كحلي+تيل، حواف، ظلال، إزالة dark mode/scheme).
- تحميل خط `IBM Plex Sans Arabic` في التطبيقات الثلاثة وإزالة Cairo + الخطوط غير المستخدمة (tajawal).
- توحيد المكوّنات المكرّرة (LiveMap ~450 سطر، Feedback, Icon, Loader...) في `packages/shared` (تقليل عدد الملفات).
- تحديث `app.json` (splash/notification colors) وإزالة بقايا الألوان القديمة (navy/gold).
- **المخرج:** نظام تصميم موحّد واحد، وتطبيقات تُبنى بلا أخطاء.

### المرحلة 2 — إعادة بناء الشاشات على Stitch
- **الطالب أولاً** (شاشة بشاشة، بنفس ترتيب Stitch): الرئيسية خريطة-أولاً + شيت + تبويب Rafeeq AI، اختيار الفئة، السجل، المحفظة، الإعدادات، الاشتراك، المفقودات.
- ثم **الكابتن**: لوحة القيادة (متصل/التقييم/الدخل/طلب جديد)، السجل، المحفظة، الإعدادات، الملف.
- ثم **الإدارة**: Sidebar يمين + التحليلات/الطلاب/الكباتن/الرحلات/المالية.
- **خريطة عصرية تفاعلية** (ستايل مخصّص + مسار + دبابيس + ETA + حركة سلسة).
- **مساحات إعلانية** في الأماكن المدروسة (بلا تفعيل محتوى بعد).
- **المخرج:** ثلاثة تطبيقات مطابقة لـ Stitch بصرياً ووظيفياً.

### المرحلة 3 — محرّك التسعير والزون والعمولة
- محرّك تسعير مبني على المسافة (Haversine/الخرائط) + حدّ أدنى + تعرفة ليلية + ذروة بسقف.
- عمولة قابلة للضبط من لوحة التحكم + **شاشة ملخّص أرباح الكابتن** + معاينة الربح قبل القبول.
- سعر طالب موحّد (جامعة↔بيت) عبر الاشتراك ومصفوفة الأزواج.
- Seed كل محافظات الأردن وجامعاتها (إربد مفعّلة، الباقي جاهز للتفعيل).
- إدارة الأسعار/المناطق/العمولة من لوحة التحكم.
- **المخرج:** أسعار منطقية متدرّجة بالمسافة + جاهزية توسّع بكبسة زر.

### المرحلة 4 — إغلاق كل الثغرات الأمنية (تحقّق فعلي)
- تدقيق فعلي لكل ثغرة موثّقة سابقاً والتأكّد من إغلاقها: قنوات البث الخاصة، حجوزات المحفظة اليتيمة، تجاوز MFA عبر OTP، zero-sum الكوبونات، SOS.
- فحص شامل جديد (authz على كل endpoint، rate limiting، إدخالات، رفع الملفات، أسرار env).
- رفع **PHPStan** لمستوى أعلى وجعله **بوابة CI** (كان 242 ملاحظة غير مُلزمة)، وتوحيد **PHP 8.4** في كل مكان (composer + CI).
- **المخرج:** لا ثغرة معروفة، وبوابات جودة مُلزمة في CI.

### المرحلة 5 — التكامل والصحّة (لا شيء يكسر التطبيق)
- تشغيل فعلي: `composer install` + `phpunit` (التحقّق من الـ166 اختبار المُدّعاة) + `typecheck` للتطبيقات الثلاثة.
- فحص تطابق `api-client` مع مسارات الـ backend (عقود الطلب/الاستجابة).
- فحص المسارات المعطّلة/الفارغة، حالات الخطأ، الشبكة الضعيفة، غياب المفاتيح (fallback).
- **المخرج:** تقرير صحّة يؤكّد استقرار التطبيقات الثلاثة وتكاملها.

### المرحلة 6 — AI والمزايا الجديدة والإعلانات
- ربط GPT API حقيقي + ضبط الميزانية + تفعيل مزايا AI.
- مزايا جديدة: «شارك رحلتي»، وضع الطالبات/كابتن-أنثى، الكاربول (طالبات بسياراتهن)، الرحلات المجدولة.
- تفعيل **لوحة تحكم الإعلانات** (حملات/أماكن/جدولة/إحصاءات).
- **المخرج:** تجربة ذكية متكاملة + قناة ربح إعلاني.

### المرحلة 7 — صلابة الإطلاق (Launch Hardening)
- التكاملات الحقيقية (CliQ, FCM, WhatsApp OTP, Maps, OpenAI) + أسرار الإنتاج.
- النشر (خادم/دومين/SSL/Docker) + بناء المتاجر (EAS) + الأداء + النسخ الاحتياطي + المراقبة.
- المراجعة القانونية (الشروط/الخصوصية) + تثبيت اسم العلامة ([09-BRAND-NAMING](./09-BRAND-NAMING.md)).
