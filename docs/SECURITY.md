# مراجعة الأمان — رفيق (Rafeeq)

> مراجعة أمنية صادقة لوضع المشروع. تُحدَّث مع كل تغيير أمني.

## ✅ ضوابط أمنية مطبَّقة (Backend)
- **المصادقة**: Laravel Sanctum + رموز UUID + إبطال عند 401.
- **الصلاحيات**: RBAC (أدوار/صلاحيات) + middleware `role:` و`permission:` على كل مسارات الإدارة.
- **OTP**: مُخزّن مُجزّأً (hashed) + TTL + حد محاولات + cooldown + إبطال السابق.
- **Throttling**: `throttle:auth` على auth/otp (6/دقيقة per-phone) + **`throttle:api` عام (120/دقيقة per-user/IP)** على كل مسارات الـAPI + **`throttle:sensitive` (20/دقيقة)** على المسارات الحساسة (التحقق من الكوبونات).
- **رؤوس أمنية (`SecurityHeaders` middleware)**: `X-Content-Type-Options: nosniff` · `X-Frame-Options: DENY` · `Referrer-Policy` · `Permissions-Policy` · `Content-Security-Policy` صارم (JSON-only) · **HSTS عند HTTPS فقط**.
- **التدقيق**: `audit_logs` لكل عملية حساسة (دفع/رحلة/أمان...).
- **حماية البيانات**: `$fillable` على كل النماذج (منع mass-assignment)، استعلامات Eloquent مُعاملة (parameterized) — لا حقن SQL.
- **مفاتيح UUID** (لا تعداد تسلسلي للموارد).
- **رفع الملفات**: التحقق من النوع (صور فقط) + الحجم (8MB) + التخزين على disk خاص `secure` (يُقدَّم عبر روابط مؤقتة).
- **الأموال**: تمرّ دائماً عبر المنصة (لا كاش مباشر) + idempotency في الفوترة والدفع.
- **ملكية الموارد**: نقاط الدفع للطالب تتحقق من `assertOwner`؛ دفع الاشتراك يتحقق `student_id`.

## ⚠️ ملاحظات/مخاطر معروفة وخطة المعالجة
1. **تخزين توكن الإدارة في `localStorage`** (لوحة Next.js): نمط شائع للـ SPA، لكنه عرضة لـ XSS. الخطة: الانتقال لمصادقة كوكي httpOnly (Sanctum SPA) قبل الإطلاق — **M10**.
2. **`apiUrl` الافتراضي `http://localhost`**: للتطوير فقط. الإنتاج يفرض **HTTPS** + HSTS — **M10**.
3. **تبديل RTL لحظياً**: `I18nManager.forceRTL` يتطلب إعادة تشغيل التطبيق على الأجهزة الحقيقية (قيد React Native). الخطة: مطالبة بإعادة التشغيل عند تبديل اللغة، أو تخطيط ثنائي الاتجاه عبر الستايل — **M3/M5**.
4. **مفاتيح التكاملات**: OpenAI/Firebase/CliQ/SMS تُقرأ من `.env` (غير مُتعقّبة في git). يجب إدارتها عبر Secrets Manager في الإنتاج — **M10**.

## 📦 ثغرات npm (تقرير صادق)
`npm audit` يُظهر ~40 تنبيهاً (1 critical, 18 high, ...). الغالبية العظمى **تبعيات غير مباشرة (transitive)** ضمن Expo SDK 51 و Next.js 14 و أدوات البناء، وكثير منها **dev-only** غير مُعرَّض في تطبيق منشور.
- **لا تُشغّل `npm audit fix --force`** — سيرقّي حزماً لإصدارات كاسرة (breaking) ويُعطّل البناء (Expo/Next حسّاسة للإصدارات).
- **النهج الصحيح**: ترقية **Expo SDK** و**Next.js** إلى أحدث إصدار ثابت (يجلب التصحيحات الأمنية للتبعيات تلقائياً)، ومراجعة `npm audit` بعدها. يُنفَّذ ضمن صلابة الإطلاق **M10**.
- التطبيقات (RN/SPA) لا تُنفِّذ كوداً من المستخدم، ما يقلّل سطح الخطر لمعظم هذه التنبيهات.

## 🔧 إصلاحات هذه الدفعة

- لوحة الإدارة: `getLocale` صار يقرأ تفضيل اللغة (Accept-Language صحيح للنصوص الخادمية).

## نموذج الأمان (Security Model)

# نموذج الأمان (Security Model)

> الأمان أهم قسم في رفيق — فلسفة التصميم: **Trust First**.

## طبقات الأمان (Defense in Depth)
| الطبقة | الوصف | الحالة |
|--------|--------|--------|
| 1 | **OTP Verification** — تحقق من الهاتف عند التسجيل/الدخول | Phase 1 |
| 2 | **توثيق السائق** — مراجعة بيانات الكابتن | Phase 1 |
| 3 | **رفع الوثائق الرسمية** — رخصة/هوية/تأمين/عدم محكومية | Phase 1 |
| 4 | **Face Verification** — مطابقة الوجه (نتيجة فقط) | Phase 1/5 |
| 5 | **Liveness Detection** — كشف الحيوية | Phase 1/5 |
| 6 | **Trip OTP** — كود لبدء الرحلة (طالب↔كابتن) | Phase 2 |
| 7 | **SOS Emergency System** — زر طوارئ | Phase 7 |
| 8 | **Audit Logs** — سجل كامل للعمليات الحساسة | Phase 1 |
| 9 | **Risk Monitoring** — مراقبة الأنماط الخطرة بالـ AI | Phase 5 |

## المصادقة والصلاحيات
- **المصادقة:** Laravel Sanctum (Bearer tokens) للموبايل والويب.
- **OTP:** يُخزّن hashed، صلاحية 5 دقائق، حد أقصى 5 محاولات، cooldown 60 ثانية لإعادة الإرسال.
- **RBAC:** أدوار (student/driver/support/supervisor/admin) + صلاحيات دقيقة لكل عملية.
- **Rate Limiting:** على مسارات OTP والدخول (throttle) لمنع الـ brute force.
- **كلمات المرور:** Argon2id/bcrypt، مع سياسة قوة دنيا.

## حماية البيانات
- تشفير الحقول الحساسة (هوية وطنية، وثائق) عبر `encrypted` cast.
- **البيانات الحيوية:** لا نخزّن صور الوجه/الـ liveness — فقط نتيجة (verified/at). يُفضّل استخدام مزوّد KYC خارجي.
- HTTPS إلزامي، HSTS، أعلام كوكيز آمنة.
- مبدأ أقل صلاحية (Least Privilege) في كل الوصول.

## نظام SOS
عند الضغط على زر الطوارئ: إرسال الموقع + بيانات الرحلة → فتح حادثة طوارئ → إشعار الإدارة فوراً.

## الشكاوى — مستويات الخطورة
| المستوى | أمثلة | الإجراء |
|---------|-------|---------|
| منخفض | تأخير | تنبيه/متابعة |
| متوسط | سلوك غير مناسب | تحذير + تسجيل |
| مرتفع | قيادة خطرة | مراجعة عاجلة |
| حرج | تحرش/عنف/تهديد | **تجميد الحساب فوراً + فتح تحقيق + إشعار الإدارة** |

## التدقيق (Audit)
كل عملية حساسة (دخول، موافقة كابتن، اعتماد دفعة، تجميد حساب، تغيير صلاحية) تُسجَّل في `audit_logs`
مع الفاعل والوقت والـ before/after والـ IP.

## الامتثال (Compliance)
- الالتزام بقوانين حماية البيانات في الأردن.
- إقرار خصوصية + شروط استخدام واضحة (خصوصاً لخدمة إرسال الأغراض).
- سياسة احتفاظ بالبيانات (Data Retention) وحق المستخدم في الحذف.
