# مراجعة الأمان — رفيق (Rafeeq)

> مراجعة أمنية صادقة لوضع المشروع. تُحدَّث مع كل تغيير أمني.

## ✅ ضوابط أمنية مطبَّقة (Backend)
- **المصادقة**: Laravel Sanctum + رموز UUID + إبطال عند 401.
- **الصلاحيات**: RBAC (أدوار/صلاحيات) + middleware `role:` و`permission:` على كل مسارات الإدارة.
- **OTP**: مُخزّن مُجزّأً (hashed) + TTL + حد محاولات + cooldown + إبطال السابق.
- **Throttling**: على مسارات auth/otp.
- **التدقيق**: `audit_logs` لكل عملية حساسة (دفع/رحلة/أمان...).
- **حماية البيانات**: `$fillable` على كل النماذج (منع mass-assignment)، استعلامات Eloquent مُعاملة (parameterized) — لا حقن SQL.
- **مفاتيح UUID** (لا تعداد تسلسلي للموارد).
- **رفع الملفات**: التحقق من النوع (صور فقط) + الحجم (8MB) + التخزين على disk خاص `secure` (يُقدَّم عبر روابط مؤقتة).
- **الأموال**: تمرّ دائماً عبر المنصة (لا كاش مباشر) + idempotency في الفوترة والدفع.
- **ملكية الموارد**: نقاط الدفع للطالب تتحقق من `assertOwner`؛ دفع الاشتراك يتحقق `student_id`.

## ⚠️ ملاحظات/مخاطر معروفة وخطة المعالجة
1. **تخزين توكن الإدارة في `localStorage`** (لوحة Next.js): نمط شائع للـ SPA، لكنه عرضة لـ XSS. الخطة: الانتقال لمصادقة كوكي httpOnly (Sanctum SPA) قبل الإطلاق — **M10**.
2. **`apiUrl` الافتراضي `http://localhost`**: للتطوير فقط. الإنتاج يفرض **HTTPS** + HSTS — **M10**.
3. **تبديل RTL لحظياً**: `I18nManager.forceRTL` يتطلب إعادة تشغيل التطبيق على الأجهزة الحقيقية (قيد React Native). الخطة: مطالبة بإعادة التشغيل عند تبديل اللغة، أو تخطيط ثنائي الاتجاه عبر الستايل — **M3/M5**.
4. **مفاتيح التكاملات**: OpenAI/Firebase/CliQ/SMS تُقرأ من `.env` (غير مُتعقّبة في git). يجب إدارتها عبر Secrets Manager في الإنتاج — **M10**.

## 📦 ثغرات npm (تقرير صادق)
`npm audit` يُظهر ~40 تنبيهاً (1 critical, 18 high, ...). الغالبية العظمى **تبعيات غير مباشرة (transitive)** ضمن Expo SDK 51 و Next.js 14 و أدوات البناء، وكثير منها **dev-only** غير مُعرَّض في تطبيق منشور.
- **لا تُشغّل `npm audit fix --force`** — سيرقّي حزماً لإصدارات كاسرة (breaking) ويُعطّل البناء (Expo/Next حسّاسة للإصدارات).
- **النهج الصحيح**: ترقية **Expo SDK** و**Next.js** إلى أحدث إصدار ثابت (يجلب التصحيحات الأمنية للتبعيات تلقائياً)، ومراجعة `npm audit` بعدها. يُنفَّذ ضمن صلابة الإطلاق **M10**.
- التطبيقات (RN/SPA) لا تُنفِّذ كوداً من المستخدم، ما يقلّل سطح الخطر لمعظم هذه التنبيهات.

## 🔧 إصلاحات هذه الدفعة
- مواءمة إصدار TypeScript مع المتوقّع من Expo (`~5.3.3`) لإزالة تحذير التوافق.
- لوحة الإدارة: `getLocale` صار يقرأ تفضيل اللغة (Accept-Language صحيح للنصوص الخادمية).
