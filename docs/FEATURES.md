> نظرة واحدة على **كل** ما بُني في المنصّة من كل النواحي: الوحدات الخلفية، الواجهات، الأمان، الدفع، والاختبارات.
> آخر تحديث: بعد دمج كل الشغل على `foundation/phase-0-1` (حتى RFQ-150).
> للحالة التفصيلية والخطوة التالية: `docs/PROGRESS.md`. للخطة الكاملة: `docs/ROADMAP.md` + `docs/EXECUTION_PLAN.md`.

## 2. النواة (Core / Shared / Infrastructure)

- **Core**: `ApiResponse` موحّد، `BaseController` (ok/created)، `BaseService` (transaction)، `BaseRepository`، استثناءات المجال (`BusinessRuleException`→422، `AuthorizationException`→403)، Middleware (RBAC، تدقيق الطلبات، فرض JSON، تحديد اللغة).
- **RBAC + Audit**: أدوار وصلاحيات (Role/Permission/HasRoles) — `admin` متجاوز للصلاحيات (superuser) + سجل تدقيق لكل عملية حساسة.
- **Shared/Enums**: تعدادات بـ `labelAr()` (نوع المستخدم، حالة الرحلة، نوع حركة المحفظة، …).
- **Infrastructure/Gpt**: عميل OpenAI (chat + vision) + `NullGptClient` (بديل آمن بلا مفتاح → مراجعة يدوية).
- **Infrastructure/Push**: `FcmPushGateway` (Firebase HTTP v1) + `LogPushGateway` بديل.
- **Infrastructure/Sms**: بوابة SMS HTTP + بديل log.

### المصادقة الثنائية (MFA / TOTP) — RFQ-147/149
- تطبيق **TOTP وفق RFC 6238** مكتفٍ ذاتياً (بلا أي مكتبة خارجية): `TotpService` (توليد سر Base32، رابط `otpauth://` للـ QR، تحقّق مع تحمّل انحراف الساعة ±خطوة).
- `MfaService`: تسجيل تدريجي (`beginSetup` → `confirmSetup`) + **رموز استرداد لمرة واحدة** + **تحدّي دخول قصير العمر** عبر cache + إيقاف.
- عند تفعيل MFA: الدخول بكلمة المرور **لا يصدر توكن** — يرجع `mfa_required` + `mfa_token`، ويُكمَل عبر `POST /auth/mfa/verify` برمز TOTP أو رمز استرداد.
- الإدارة (مصادَق): `/auth/mfa/setup` · `/confirm` · `/disable` — **لحسابات الموظفين فقط**.
- أعمدة `users` مشفّرة: `mfa_secret` + `mfa_enabled_at` + `mfa_recovery_codes`.
- **اختبارات**: 7 Feature + 5 Unit.

### نموذج الرحلة
- **التجميع الزوني باب-لباب**: الطلاب في نفس الزون + نفس الجامعة + نافذة زمنية متقاربة → رحلة واحدة بكابتن واحد (ترتيب التقاط مُحسّن).
- **سيارات خاصة لا باصات**: السعة الافتراضية 4، الأقصى ~7.
- **Express (مستعجل)**: رحلة خاصة/أولوية برسوم + surge ضمن سقف عادل + معاينة أرباح الكابتن.

### حدود المناطق المضلّعة (Geofence) — RFQ-148/149
- عمود `zones.boundary` (مصفوفة JSON من رؤوس `[lat,lng]`) كحدّ جغرافي اختياري.
- `Zone::containsPoint()` بخوارزمية **ray-casting** (نقطة داخل مضلّع) + `hasBoundary` + `withinRadius`.
- `ZoneService::nearest()` يحلّ بالترتيب: **احتواء المضلّع → دائرة النطاق → أقرب مركز** (متوافق رجعياً).
- تحقّق من الحدود (3 رؤوس على الأقل + نطاقات lat/lng صحيحة). **3 اختبارات Feature**.

## 5. المال (الدفع + المحفظة + العمولة + السحب)

| الوحدة | الوظيفة |
|--------|---------|
| **Payments** | طلبات دفع CliQ (رقم `RFQ-YYYY-#####`) + رفع إشعار التحويل + **تحقّق GPT Vision** → اعتماد تلقائي/مراجعة يدوية → تفعيل (شحن محفظة/اشتراك) |
| **Wallet** | محفظة مسبقة الدفع بالفلس + حركات موقّعة + **حجز رصيد (holds)** + قفل صفّي آمن |
| **Payouts** | **سحب أرباح الكابتن** + رُتب/أداء الكابتن |
| **Reports** | **التقارير المالية للإدارة** |

### دورة المال (مكافحة تسريب العمولة)
- الطالب يشحن رصيده عبر CliQ ويدفع من الرصيد/الاشتراك.
- عند **بدء الرحلة**: يُحجز السعر لكل راكب يدفع من المحفظة (المتاح = الرصيد − المحجوز).
- عند **الصعود**: يُلتقط الحجز (خصم فعلي)، **تُحجز عمولة المنصة**، ويُحاسب الكابتن (يُضاف لمحفظته).
- عند **الإلغاء**: يُحرَّر الحجز. → **الأموال تمرّ عبر المنصة دائماً، الكابتن لا يقبض كاش مباشرة**.

### سحب الأرباح + الرُتب — RFQ-141/142
- `Modules/Payouts`: طلب السحب يخصم من المحفظة فوراً، الإدارة تعتمد (paid) أو ترفض (يُعاد المبلغ). حدّ أدنى 5 د.أ.
- `/driver/performance`: الرتبة (Bronze→Silver→Gold→Platinum) + التقدّم + الأرباح المتاحة + التقييم + عدد الرحلات. **6 اختبارات**.

### التقارير المالية — RFQ-145/146
- `FinancialReportService.summary(from,to,zoneId)`: رحلات مدفوعة + إجمالي الأجور + **عمولة المنصة (الإيراد)** + أرباح الكباتن + السحوبات + الشحن + إيراد الاشتراكات، مع تفصيل **حسب الزون**.
- `GET /api/v1/admin/reports/financial` (صلاحية `analytics.view`). **3 اختبارات**.

## 6. السلامة ومكافحة الاحتيال

| الوحدة/الميزة | الوظيفة |
|---------------|---------|
| **OTP الطرفين** | كود صعود + كود إنزال داخل التطبيق؛ إنهاء رحلة بلا تأكيد إنزال يرفع علامة خطورة |
| **كشف الاحتيال عبر GPS** | (1) عدم تطابق الموقع عند الصعود؛ (2) **الرحلة الوهمية** (إلغاء + متابعة الكابتن للمسار قرب نفس الطلاب) |
| **Safety / Risk Flags** | علامات خطورة + سجلّات الإلغاء + قواعد كشف + واجهة إدارة |
| **SOS** | زر طوارئ + حوادث + تنبيه فريق السلامة + تجميد عند الحرج |
| **AI Fraud Monitor** | تسجيل خطورة لكل حساب + **كشف تواطؤ الكابتن↔الطالب** + **تجميد تلقائي عند العتبة** |
| **Dispute Center** | مركز نزاعات/تحقيقات يجمّع كل الأدلّة + سير عمل المعالجة |

### Risk Score + التجميد التلقائي + مركز النزاعات (RFQ-157/158/159)
- **كشف الأنماط المتقدّم** (`FraudMonitorService.detectCollusionFor`): يرصد التسريب خارج المنصّة عندما يُلغي الكابتن رحلات تخصّ **نفس الطالب** بشكل متكرر (≥3 خلال 30 يوم) → علامة `repeat_cancel_pairing` حرجة (idempotent). `assess()` = كشف + درجة خطر (0..100) + مستوى + الأنماط.
- **التجميد التلقائي** (`DisputeService.investigate`): عند `score ≥ 85` **أو** وجود أي علامة حرجة → يُجمَّد الحساب (Suspended + إشعار `AccountFrozen`) ويُفتح **ملف تحقيق** آلياً (idempotent لكل subject+type). `sweep()` يمرّ على أعلى الحسابات خطورة.
- **مركز النزاعات**: ملف لكل حالة يجمّع **الأدلّة** (علامات الخطر + سجل الإلغاءات + مراقبات الرحلة الوهمية GPS + درجة الخطر الحيّة). سير عمل: فتح/إسناد/معالجة (إجراء: تجميد/تحذير/تبرئة/حظر)/رفض + تجميد/رفع تجميد. مسارات `/api/v1/admin/disputes/*` (admin+supervisor). واجهة `/disputes` في لوحة الإدارة. مغطّى بـ **7 + 4 اختبارات**.

> الأساس: boarding OTP + drop-off OTP + كشف الرحلة الوهمية بالـ GPS + Risk Score + سجلّات كأدلّة في مركز النزاعات.

### بوابة ولي الأمر (Guardian Portal) — RFQ-137/138
- وحدة `Modules/Guardians` + نوع مستخدم `guardian` + دور `guardian`.
- الطالب يدير أولياء أمره (إضافة بالهاتف/العلاقة + إلغاء) — الحساب يُنشأ تلقائياً، الدخول بالهاتف+OTP.
- ولي الأمر يرى: أبناءه المرتبطين + **تتبع الرحلة الحيّ** + **سجل الوصول الآمن** + **اتصال مُقنّع بالكابتن** + **SOS بالنيابة**.
- تنبيهات الانطلاق/الوصول مربوطة بأمان داخل تدفق الرحلة. حماية: 403 لأي طالب غير مرتبط. **7 اختبارات**.

### المحادثة (Chat) — RFQ-139/140
- محادثة مرتبطة بالرحلة، تفويض لكل محادثة (الطرفان فقط، 403 لغيرهما)، بثّ `ChatMessageSent` + إشعار. شاشة في تطبيقي الطالب والكابتن. **5 اختبارات**.

## 11. المتبقّي للتشغيل الرسمي
1. **تشغيل/اختبار فعلي محلياً**: تشغيل migrations على PostgreSQL + تشغيل التطبيقات. (✅ **الفرونت مُتحقَّق type-check**: كل الحزم الست تمرّ `tsc --noEmit` بنجاح — RFQ-162→165.)
2. **تكاملات خارجية**: `OPENAI_API_KEY` (Vision)، `FIREBASE_*` (FCM)، `CLIQ_*`، `SMS_*` — كلها اختيارية (النظام يعمل عبر المراجعة اليدوية + إشعار داخل التطبيق + log بدونها).
3. تشغيل عامل الجدولة (`php artisan schedule:work` / cron) في الإنتاج — أوامر مجدولة: `prune-otps` (ساعة)، `match-rides` (5 دقائق)، **`fraud-sweep` (ساعة)**.
4. توسيع تغطية الاختبارات + تلميع نهائي.
