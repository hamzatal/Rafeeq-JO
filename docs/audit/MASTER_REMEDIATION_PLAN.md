# خطة الإصلاح الشاملة والجاهزية للإطلاق — رفيق (Master Remediation Plan)

> **المرجع التشخيصي:** `docs/audit/AUDIT_2026-07.md`.
> **الهدف:** الوصول بالمنصّة (طالب + كابتن + أدمن + باك إند) إلى **إطلاق عالمي كامل** بلا أخطاء تصميم، بأمان قوي، وباك إند يتحمّل الضغط.
> **المبدأ:** كل مرحلة قابلة للشحن وحدها (commit `[RFQ-###]` + تحديث `PROGRESS.md`)، وتُختبر على **لايت+دارك × عربي+إنجليزي** قبل الإغلاق.

## 0. القرارات التي تحتاج موافقتك أولاً (Blocking Decisions)

قبل التنفيذ، أحتاج قرارك في 3 نقاط لأنها تحدّد كل ما بعدها:

1. **لون الهوية النهائي (P0/قانوني):** نبتعد عن الليموني المطابق لـ inDrive. الخيارات المقترحة:
   - **(أ)** أخضر أردني عصري (يربط الهوية بالأردن) — مثلاً `#0EA05A`.
   - **(ب)** تركوازي/فيروزي مميّز `#12B3A6`.
   - **(ج)** لون تختاره أنت (نطبّقه كتوكن واحد عبر التطبيقات الثلاثة).
2. **هوية موحّدة واحدة:** نعتمد **نفس اللون الأساسي + التأكيد** للطالب والكابتن والأدمن (تماسك بصري)، مع اختلاف طفيف مسموح فقط في تفاصيل ثانوية. موافق؟
3. **نطاق التنفيذ الآن:** أبدأ فوراً بـ **المرحلة 1 (توحيد الهوية + إزالة inDrive)** بعد اختيارك اللون، أم تريد مراجعة الخطة كاملة أولاً؟

> بمجرد أن تحدّد اللون، أنفّذ توكن واحد في `scheme.ts` + `tailwind.config.ts` فينعكس تلقائياً على كل المكوّنات التي تقرأ من الثيم.

## 2. المرحلة 1 — توحيد الهوية البصرية + المعالجة القانونية (P0)

**الهدف:** لون واحد أصلي + هوية واحدة عبر التطبيقات الثلاثة + إزالة كل أثر «inDrive».

- [ ] **1.1** اعتماد توكن اللون النهائي (بعد قرارك) في `packages/shared/src/theme/scheme.ts` (يخدم الطالب+الكابتن).
- [ ] **1.2** مواءمة `admin-dashboard/tailwind.config.ts` + `globals.css` على **نفس** اللون (إزالة إندِيغو/سماوي v4).
- [ ] **1.3** إزالة إشارات «inDrive» من التعليقات (`scheme.ts`) والتوثيق (`PROGRESS.md` وغيره).
- [ ] **1.4** حذف `packages/shared/src/theme/colors.ts` القديم (أخضر أردني v1) أو تفريغه من الصادرات غير المستخدمة + تنظيف `index.ts`.
- [ ] **1.5** توحيد `docs/DESIGN_SYSTEM.md` كمصدر تصميم معتمد واحد + وسم `DESIGN.md` كمُؤرشَف.
- **القبول:** لون واحد يظهر في التطبيقات الثلاثة؛ صفر إشارات inDrive؛ `grep` لا يجد الليموني `#C1F11D` ولا `#4F46E5` ولا `#0B7A43`.

## 3. المرحلة 2 — الدارك/لايت والألوان الثابتة + RTL (P0/P1)

**الهدف:** كل شاشة تحترم الثيم؛ لا لون غامق على خلفية سوداء؛ RTL سليم.

- [ ] **2.1** تحويل ألوان `ErrorBoundary.tsx` (طالب+كابتن) لتقرأ من الثيم.
- [ ] **2.2** معالجة الخلفيات الغامقة الثابتة `#0E0F12` في `welcome.tsx` و`AuthShell.tsx`: إمّا توكن ثيم-واعٍ، أو تثبيت متعمّد موثّق (شاشة hero) مع ضمان تباين النص.
- [ ] **2.3** تصحيح `push.ts` (`lightColor`) و`MapBackdrop.tsx` (`routeColor`) للون البراند الجديد.
- [ ] **2.4** تدقيق بصري شاشة-بشاشة (الطالب 20+، الكابتن، اللوحة 26 صفحة) على لايت+دارك، وإصلاح أي عنصر بتباين ضعيف (< WCAG 4.5:1).
- [ ] **2.5** RTL: فرض الاتجاه مبكّراً في الإقلاع + مطالبة بإعادة التشغيل عند تبديل اللغة (رسالة واضحة).
- **القبول:** لا ألوان hex ثابتة في الشاشات (عدا HTML الطباعة/الخرائط)؛ كل شاشة سليمة في الوضعين واللغتين.

## 4. المرحلة 3 — أداء وتجربة لوحة الإدارة (P0)

**الهدف:** لا وميض، لا «قفزة» بيانات، تحميل واضح، تنقّل سريع.

- [ ] **3.1** إضافة `loading.tsx` لكل مجموعة مسارات + مكوّن `Skeleton`/`TableSkeleton`/`CardSkeleton` مشترك.
- [ ] **3.2** استبدال أيقونات Material Symbols النصّية بأيقونات SVG (مثل `lucide-react`) — يقضي على الوميض والحجب.
- [ ] **3.3** ضغط/تحسين `amman-map.jpg` (5MB → < 200KB) أو استخدام `next/image` + blur placeholder، أو استبدالها بخلفية متجهة خفيفة.
- [ ] **3.4** إدخال طبقة كاش بيانات (SWR أو React Query) مع revalidate + مشاركة الكاش بين الصفحات.
- [ ] **3.5** تحويل الصفحات القابلة للعرض الخادمي إلى RSC حيث يمكن (تقليل `'use client'`).
- [ ] **3.6** إعادة توجيه المستخدم المُصادَق بعيداً عن `/login`.
- **القبول:** كل صفحة تُظهر skeleton فوري عند الدخول؛ لا كلمات أيقونات تظهر؛ Lighthouse للأداء مقبول؛ التنقّل لا يعيد الجلب بلا داعٍ.

## 5. المرحلة 4 — إصلاحات الباك إند الحرجة (P0)

**الهدف:** سلامة الأموال والرحلات تحت التزامن.

- [ ] **4.1** `DriverTripController::acceptOffer`: `lockForUpdate` + تحديث ذرّي مشروط بالحالة (يمنع القبول المزدوج).
